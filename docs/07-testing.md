# 07 测试策略与质量保障

## 测试分层

- Unit：最小功能单元（采集、构建、检索、重排、规则判断）。
- Integration：模块协作（检索到生成的关键链路）。
- E2E：命令行全流程 smoke。

## 重点关注场景

- 新闻召回不足或被误过滤。
- 向量库不可用时的内存回退正确性。
- LLM 不可用时模板输出结构完整性。
- 报告引用编号和证据列表一致性。

## 执行

```bash
pytest
```

## 质量基线建议

- 每次改动至少覆盖对应单测。
- 涉及检索/证据逻辑变更时，增加回归样例。
- 版本发布前执行完整 integration + e2e。
